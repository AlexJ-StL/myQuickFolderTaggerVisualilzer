# myQuickFolderTaggerVisualizer

> **Brevity Note:** This documentation follows a "Signal-over-Noise" protocol. Technical accuracy and security are prioritized over narrative fluff.

## 1. Executive Summary (The "What")
A collection of Python visualization scripts designed to analyze and graphically represent codebase metadata generated by the myQuickFolderTagger tool. 
- **Core Value:** Transforms raw codebase tagging data into actionable visual insights for software architecture analysis
- **Primary Stakeholder:** Software architects, tech leads, and development teams seeking to understand technology distribution and codebase structure

## 2. First Principles & Architecture (The "How")
Explain the 2nd and 3rd order consequences of the design choices made here.
- **State Management:** Each script operates independently with functional programming principles - no shared state between visualizations
- **Data Integrity:** Uses pandas for robust CSV/dataframe handling with explicit column cleaning and NA value handling
- **Security Logic:** Scripts only read local CSV files; no network calls or external API dependencies; input validation through pandas built-in sanitization

## 3. Security & Ethics (Mandatory)
- **API/Secret Handling:** This project uses local file operations only; no API keys or secrets are required or used
- **Attack Vector Mitigation:** All file operations use relative paths; input data validation through pandas; no eval() or exec() calls
- **Ethical Bound:** This project is built to add positive value through codebase visualization. Use that results in the harm of others or betrayal of these ethics is a violation of the intended use-case.
- **Proactive Sanitization:** Before finalizing any documentation or code example, perform a mental "Secret Scan." Ensure no local paths (e.g., C:\Users\AlexJ\...), local IP addresses, or placeholder strings that look like real API keys are present.

## 5. Configuration (`config.py`)
This tool automatically calculates the Longest Common Prefix amongst all of your codebase paths and seamlessly truncates it for security and multi-root compatibility (e.g., dynamically establishing `Repos` and `.archived-repos` as separate structural containers). 

You can customize the following in `config.py`:
- **STOP_WORDS**: Words you wish to manually exclude from the Natural Language visualizations to reduce noise.
